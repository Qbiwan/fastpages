1. How to install docker on Ubuntu

  • Update Software Repositories
  • Install Docker on Ubuntu 18.04
  • If old version of Docker needs to be removed first, refer to this Guide)
sudo apt-get update
sudo apt install docker.io

Reference: How to Install Docker On Ubuntu 18.04

After you installed Docker, you could have the error message of Permission Denied. You might need to create a docker group if you dont want to use sudo everytime.

(base) qbiwan@workspace:~$ docker ps
Got permission denied while trying to connect to the Docker daemon socket at unix:///var/run/docker.sock: Get http://%2Fvar%2Frun%2Fdocker.sock/v1.40/containers/json: dial unix /var/run/docker.sock: connect: permission denied

Use the following command to create a docker group since you are not the root user. Test whether it works by running docker run hello-world

sudo groupadd docker
sudo usermod -aG docker ${USER} 
# logout of ubuntu and login again
newgrp docker
docker run hello-world

4. Build a container from one of these images

Let's say you want to build a redis container. You could do it in normal or detached mode

docker run redis  # docker run == docker pull plus docker start

docker run -d redis  # detached mode

Check if the container is running by using docker ps

CONTAINER ID        IMAGE               COMMAND                  CREATED              STATUS              PORTS               NAMES
7e1e07e7cd87        redis               "docker-entrypoint.s…"   About a minute ago   Up About a minute   6379/tcp            agitated_jennings

To stop the container, we can use its CONTAINER ID or NAMES

docker stop 7e1e07e7cd87

OR

docker stop agitated_jennings

To restart the container, use

docker start 7e1e07e7cd87

Normal or detached mode

  • docker run redis will run the container in normal mode. To kill the container just press ctrl-C. `\

  • docker run -d redis will run the container in detached mode i.e. in the background. To kill it you have to use docker stop 7e1e07e7cd87

5. Specifying Ports

Ports

You can map the port of the local computer to the port of the container using the flag -p. Foe example, this command -p 8080:8080 map port 8080 of your local computer to port 8080 of your container. Note that

  • Mapping one host port to two different container ports (NO)
  • Mapping two host ports to one container port (YES)


If you tried to map one host port to 2 container ports, you will have a conflict.However, two container ports can be listening on the same port 6379 while your host computer has two different ports.

For example, redis:latest and redis:4.0 can both be listening on port 6379, but your local ports mapped to them must be different. So you must map port 6000 to redis:latest and 6001 to redis:9.90

(base) qbiwan@workspace:~$ docker run -p6000:6379 -d redis
660ff75d97d07237d90e7c51f4b7bd16e43d594fbf612379bc6ea1651912ad61

(base) qbiwan@workspace:~$ docker run -p6001:6379 -d redis:4.0
c34abecfaf261521a4c7ad5e0b39af63963cc3ad69f1fd300a6a234258b9a919

This will not cause a conflict as we can see below

(base) qbiwan@workspace:~$ docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED              STATUS              PORTS                    NAMES

8. Access the terminal inside the container

To access the terminal inside the container we use the flag -it, where it meansinterative terminal. Below we access bash inside a container named redis-third using this command exec -it

To confirm that we successfully accessed the interactive terminal, note that the command promt changed from (base) qbiwan@workspace:~$ to root@e1cf0aa306db:/data in this case.

(base) qbiwan@workspace:~$ docker exec -it redis-third bash
root@e1cf0aa306db:/data#

print the current directory we are in, and we can see that we are at /data

root@e1cf0aa306db:/data# pwd
/data

Change to parent directory and we find ourselves at root /s

root@e1cf0aa306db:/data# cd ..
root@e1cf0aa306db:/# pwd
/

Let's list all the directories here

root@e1cf0aa306db:/# ls
bin  boot  data  dev  etc  home  lib  lib64  media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var

Type exit to get out of the interactive terminal and get back to our local computer

root@e1cf0aa306db:/# exit
exit

Thereafter,you start the MongoDB container on your local computer and start working on your python app, which uses the database. Once done, push the python codes to github, which will in turn do continuous integration with your private company server, where your python image will be stored.

MongoDB

Now we create the MongoDB container by explicitly specifying this newly-created network with the command flag -net mongo-network, as well as other configurations as follows:

  • port : -p 27017:27017
  • name of this container : --name mongodb
  • username to the Mongo database : -e MONGO_INITDB_ROOT_USERNAME=admin # not recommended
  • password to the Mongo database e MONGO_INITDB_ROOT_PASSWORD=password # not recommended
  • docker network to join on this computer -net mongo-network
  • image used mongo 
(base) qbiwan@workspace:~$ docker run -d \
> -p 27017:27017 \
> --name mongodb \
> -e MONGO_INITDB_ROOT_USERNAME=admin \
> -e MONGO_INITDB_ROOT_PASSWORD=password \

Mongo-express

Next we create mongo-express, the database admin-user interface,so that we can administer the mongo-db conveniently with a friendly user-interface. We will use the following configuration. Note that the container will also be inside the new docker network mongo-network we just created.

  • port : -p 8081:8081
  • specify the username for mongo-db : -e ME_CONFIG_MONGODB_ADMINUSERNAME=admin
  • specify the password for mongo-db : -e ME_CONFIG_MONGODB_ADMINPASSWORD=password
  • the docker network to be a part of : --net mongo-network
  • the name of this container : --name mongo-express
  • the mongo database that this mongo-express container will be administering : -e ME_CONFIG_MONGODB_SERVER=mongodb
  • the image for this container : mongo-express

10. Docker Compose

Instead of writing all the commands above, we can do it with docker-compose in a single yaml file. Create a yaml file called mongo-docker-compose.yaml with the following content:

version:'3'                                       # docker version 
services:
  mongodb:                                        # 1st container name
    image: mongo                                  # which image to build from
    ports:
      - 27017:27017                               # which ports to map host:container
    environment:                                  # environment variables
      - MONGO_INITDB_ROOT_USERNAME=admin          
      - MONGO_INITDB_ROOT_PASSWORD=password
  mongo-express:                                  # 2nd container name   
    image: mongo-express                          # which image to build from
    ports:
      - 8081:8081                                 # which ports to map host:container
    environent:                                   # environment variables
      - ME_CONFIG_MONGODB_ADMINUSERNAME=admin
      - ME_CONFIG_MONGODB_ADMINPASSWORD=password
      - ME_CONFIG_MONGODB_SERVER=mongodb

11. Build your own Docker image

You have to write your own Dockerfile if you want to build your own docker image. A simple Dockerfile would read like the following.

FROM node:13-alpine
    
ADD environment.yml /    

ENV MONGO_DB_USERNAME=admin \
    MONGO_DB_PWD=password
    
RUN MKDIR -p /home/app

COPY . /home/app

CMD ["node", "server.js"]

Dockerfile commands

The commands used above are as followd:

  • FROM

    • install base images, for exampleFROM tensorflow/tensorflow:2.0.1-gpu or FROM continuumio/miniconda3:4.5.11

  • ENV

    • set the environment variables. For examples ENV FLASK_APP=app.py sets FLASK_APP to be app.py. You could set the environment variables here or in docker-compose, though it is way easier to change the environment variables if you set it in Docker-Compose.yml. This is because once an image is created from a Dockerfile, it cannot be changed. As such, you will need to create build a new image, which is more tedious.

  • RUN

    • run linux commands INSIDE the container. For example RUN mkdir -p /home/app creates a directory recursively in the container (not on your local computer).

  • COPY

    • If you’re copying in local files to your Docker image, always use COPY because it’s more explicit. For example, COPY . . copies all files from current dir on your host computer to the home dir on the container. In contrast, if you use RUN copy dirA dirB the files in dirA inside the container gets copied to dirB inside the container.

  • CMD

    • CMD is the entry point command that once the container is built and ready this command will be executed. For example, CMD ["streamlit", "run", ", "app2.py"] will be the first command the containre run once built. Note that this entry-point command can be overwritten during docker run. For example docker run my-app:1.0 env will output all the env variables instead of running app2.py as follows:
